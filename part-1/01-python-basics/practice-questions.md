# Practice Questions

1. Which of the following are operators, and which are values?

- `*`
- `'hello'`
- `-88.8`
- `/`
- `-`
- `5`

**Answer:** The following operators and values are respectively:

- `*`, `/` and `-`
- `'hello'`, `-88.8` and `5`

2. Which of the following is a variable, and which is a string?

- `spam`
- `'spam'`

**Answer:** The variable is the `spam` while the `'spam'` is a literal string value.

3. Name three data types

**Answer:** The three data types mentioned in the book about Python are: `integer`, `floats` and `string`

4. What is an expression made up of? What do all expressions do?

**Answer:** And expression is made up of multiple operations and what it does is evaluate or produce a value

5. This chapter introduced assignment statements, like spam = 10. What is the difference between an expression and a statement?

**Answer:** An expression is a code that evaluates to a value, while a statement performs an action like `spam = 10`

6. What does the variable bacon contain after the following code runs?

- `bacon = 20`
- `bacon + 1`

**Answer:** The `bacon` variable still consists of `20` value because it was never reassigned after adding `1`

7. What should the following two expressions evaluate to?

- `'spam' + 'spamspam'`
- `'spam' * 3`

**Answer:** The following result would be identical shown below:

- `'spamspamspam'`
- `'spamspamspam'`

8. Why `eggs` is a valid variable name while `100` is invalid?

**Answer:** Because it is one of the Python rules regarding the variable names, it must start with any letters and underscore but not numbers

9. What three functions can be used to get the integer, floating-point number, or string version of a value?

**Answer:** The three functions to get the corresponding values respectively are: `int()`, `float()` and `str()`

10. Why does this expression cause an error? How can you fix it?

- `'I eat ' + 99 + ' burritos'`

**Answer:** The expression would throw an error because we cannot simply concatenate a string with a number in Python, and we can fix this by calling `str(99)` to convert it to string
